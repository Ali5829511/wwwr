# تحليل التصميم الجديد - صفحة الوحدات السكنية

## الأعمدة الجديدة المطلوبة (من الصورة)

حسب الصورة المرفقة، الأعمدة المطلوبة هي:

| الترتيب | اسم العمود | الوصف |
|---------|-----------|-------|
| 1 | **الرقم** | رقم تسلسلي |
| 2 | **الوحدة السكنية** | اسم الوحدة (مثل: شقة 01 عمارة 1) |
| 3 | **فئة عمارة** | نوع المبنى (قديم/جديد/فلة) |
| 4 | **رقم الجوال** | رقم جوال الساكن |
| 5 | **رقم الموقف** | رقم موقف السيارة المخصص |
| 6 | **وصف المبنى** | وصف إضافي للمبنى |
| 7 | **حالة الشقة** | حالة الإشغال (مشغول/شاغر) |
| 8 | **رقم المبنى** | رقم المبنى |
| 9 | **رقم الشقة** | رقم الشقة |
| 10 | **نوع الوحدة** | نوع الوحدة (شقة/فلة) |
| 11 | **اسم الساكن** | اسم الساكن الحالي |

## الإحصائيات المطلوبة

1. **المباني الفردية** - عدد الفلل
2. **المباني الشاغرة** - عدد الوحدات الشاغرة
3. **المباني المستغولة** - عدد الوحدات المشغولة (المستأجرة)
4. **إجمالي الشقق** - إجمالي عدد الوحدات
5. **المباني الجديدة** - عدد المباني الجديدة

## البحث والتصفية

- **البحث**: بالاسم أو رقم المبنى أو رقم الشقة أو اسم الساكن
- **حالة الشقة**: تصفية حسب جميع الحالات
- **نوع المبنى**: تصفية حسب جميع الأنواع

## الحقول المطلوبة في قاعدة البيانات

```sql
- id (المعرف)
- unit_name (الوحدة السكنية)
- building_category (فئة عمارة: قديم/جديد/فلة)
- resident_phone (رقم الجوال)
- parking_number (رقم الموقف)
- building_description (وصف المبنى)
- occupancy_status (حالة الشقة: مشغول/شاغر)
- building_number (رقم المبنى)
- unit_number (رقم الشقة)
- unit_type (نوع الوحدة: شقة/فلة)
- resident_name (اسم الساكن)
```

## التعديلات المطلوبة

1. ✅ تغيير اسم الصفحة من "إدارة الشقق" إلى "إدارة الوحدات السكنية"
2. ✅ تحديث الأعمدة في الجدول حسب القائمة أعلاه
3. ✅ تحديث الإحصائيات
4. ✅ تحديث خيارات البحث والتصفية
5. ✅ إضافة حقول جديدة في قاعدة البيانات
6. ✅ ربط بيانات الساكن (الاسم والجوال) من جدول residents
7. ✅ ربط رقم الموقف من جدول parking_spaces
